**样式**

- 和其他按钮样式统一

### 3. 更改密码页面 ✅

**页面结构**
- 页面标题：更改密码
- 输入旧密码
- 输入新密码（带密码强度检查）
- 确认新密码（带密码强度检查）
- 提交保存按钮
- 取消按钮
- 忘记密码按钮

**功能**
- 验证旧密码是否正确
- 验证新密码和确认密码是否一致
- 验证新密码强度
- 保存新密码到 localStorage
- 更新本地用户列表

**样式**
- 淡蓝色主题
- 渐变背景 + 蓝色边框
- 圆角设计
- 和全站统一

### 4. 忘记密码功能 ✅

**入口**
- 在更改密码页面底部
- 点击【忘记密码】按钮

**功能流程**
- 显示当前登录账号的邮箱
- 假邮箱验证（只要账号匹配即可通过）
- 通过后可直接重置新密码
- 输入新密码（带密码强度检查）
- 确认新密码（带密码强度检查）
- 提交保存

**实现方式**
- 目前使用本地模式实现
- 不依赖后端
- 后续可一键切换到 Supabase 模式

### 5. 返回按钮保留 ✅

## 🔧 技术实现

### 新增字段
- 学校、学院、专业字段添加到 currentUser 对象
- 保存到 localStorage
- 更新本地用户列表

### 更改密码
- 验证旧密码是否正确
- 验证新密码和确认密码是否一致
- 验证新密码强度
- 更新密码到 localStorage
- 更新本地用户列表

### 忘记密码
- 显示当前登录账号的邮箱
- 假邮箱验证（本地模式）
- 直接重置新密码
- 更新密码到 localStorage
- 更新本地用户列表

### 密码强度检查
- 实时检查密码强度
- 显示密码强度条（弱、中、强）
- 颜色：红色（弱）、橙色（中）、绿色（强）

### 数据存储
- 使用 localStorage 存储数据
- currentUser 对象包含：email、nickname、avatar、gender、birthday、school、college、major、bio、password
- 更新时同步更新 localUsers 数组

## 💡 使用说明

### 修改个人资料
1. 登录账号
2. 点击导航栏的用户头像或昵称
3. 进入个人资料页面
4. 填写学校、学院、专业
5. 点击【保存】按钮
6. 信息保存成功

### 更改密码
1. 进入个人资料页面
2. 点击【更改密码】按钮
3. 输入旧密码
4. 输入新密码
5. 确认新密码
6. 点击【提交】按钮
7. 密码修改成功

### 忘记密码

2. 点击【忘记密码】按钮

4. 输入新密码
5. 确认新密码
