# MY SANDA 校园网站 - 完整操作步骤

按顺序做完下面步骤，网站就能正常运行。

---

## 第一步：在 Supabase 里配置（约 5 分钟）

1. 打开 https://supabase.com ，登录你的账号
2. 进入你的项目（hsgvzrnvahkeujragiaz）

### 1.1 开启邮箱验证
- 左侧点 **Authentication** → **Providers** → **Email**
- 打开 **Confirm email**（必须开启）
- 点 **Save**

### 1.2 配置 URL（用于忘记密码）
- 左侧点 **Authentication** → **URL Configuration**
- **Site URL** 填：`http://localhost:3000`（本地测试用）
- **Redirect URLs** 点 **Add URL**，添加：
  - `http://localhost:3000/**`
  - 如果你已有 Netlify 地址，再加：`https://你的网站.netlify.app/**`
- 点 **Save**

### 1.3 执行建表 SQL
- 左侧点 **SQL Editor** → **New query**
- 打开项目里的 `supabase-schema.sql`，复制全部内容
- 粘贴到 SQL 编辑器
- 点 **Run**
- 看到绿色 **Success** 表示成功（如提示 policy 已存在，可忽略或先执行之前给你的「删除策略」SQL）

---

## 第二步：本地运行网站（约 2 分钟）

1. 确保项目文件夹里有：`index.html`、`css`、`js` 等
2. 用本地服务器运行（不要用 `file://` 直接打开）：
   - **方式一**：VS Code 安装 Live Server，右键 `index.html` → Open with Live Server
   - **方式二**：在项目根目录运行 `npx serve`，浏览器访问提示的地址（如 http://localhost:3000）
   - **方式三**：双击运行 `start-server.bat`（如果有）

---

## 第三步：测试功能（约 3 分钟）

1. 打开网站，点击 **注册**
2. 输入：邮箱、昵称、密码 → 提交
3. 去邮箱收验证邮件，点击链接完成验证
4. 回到网站，用邮箱 + 密码 **登录**
5. 登录后测试：发布校园生活、失物招领等

---

## 第四步：部署到 Netlify（约 3 分钟）

1. 把项目推送到 GitHub
2. 打开 https://netlify.com ，用 GitHub 登录
3. 点 **Add new site** → **Import an existing project**
4. 选择你的 GitHub 仓库
5. **Build settings** 保持默认（无需构建）
6. **Publish directory** 留空或填 `.`
7. 点 **Deploy**

---

## 第五步：上线后改 Supabase 地址（必做）

部署完成后，你的网站会有一个类似 `https://xxx.netlify.app` 的地址。

1. 打开 Supabase → **Authentication** → **URL Configuration**
2. **Site URL** 改成：`https://你的网站.netlify.app`
3. **Redirect URLs** 中已有 `http://localhost:3000/**` 可以保留
4. 再添加：`https://你的网站.netlify.app/**`
5. 点 **Save**

完成以上设置后，网站即可正常使用。

## 常见问题

| 问题 | 解决办法 |
|------|----------|
| 收不到验证邮件 | 查垃圾邮箱；Supabase 免费版有发送限制 |
| 登录提示邮箱未验证 | 去邮箱点验证链接 |
| 忘记密码链接点开没反应 | 检查 Supabase 的 Redirect URLs 是否包含你的网站地址 |
| 页面空白 | 用 http://localhost:xxx 访问，不要用 file:// |
| 登录/注册失败 | 去 Supabase → Project Settings → API，确认 anon key 是否与 `supabase-init.js` 里一致 |
